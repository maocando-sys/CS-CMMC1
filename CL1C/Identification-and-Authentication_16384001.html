CMMC LVL 1 CS : Identification and Authentication

Created by Manuel Ocando, last modified on Feb 03, 2026

1. Purpose

This policy establishes requirements to ensure that organizational users are uniquely identified and properly authenticated before accessing systems that store, process, or transmit Federal Contract Information (FCI), in accordance with CMMC Level 1 (FAR 52.204-21).

2. Scope

This policy applies to:

  • All users (employees, contractors, third parties)

  • All systems handling FCI

  • All access methods, including local and remote access

3. Policy Statements

3.1 Unique Identification

Each user accessing organizational systems is assigned a unique identifier. User identities are not shared between individuals.

3.2 Authentication

Users are authenticated using approved authentication mechanisms prior to accessing systems that handle FCI.

Authentication credentials are protected from unauthorized disclosure and misuse.

3.3 Credential Management

Authentication credentials are:

  • Issued to individual users

  • Changed or revoked when access is no longer required

  • Managed in accordance with documented procedures

4. Roles and Responsibilities

  • System Administrators implement authentication controls

  • Users safeguard credentials and report compromise

5. Enforcement

Unauthorized use of credentials or circumvention of authentication controls may result in access revocation or disciplinary action.

6. Compliance Mapping

  • IA.L1-3.5.1 – Identify system users

  • IA.L1-3.5.2 – Authenticate system users

7. Review

Reviewed annually or upon significant change.
