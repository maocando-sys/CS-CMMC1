CMMC LVL 1 CS : 4. CMMC Level 1 Controls

Created by Manuel Ocando, last modified on Feb 03, 2026

Purpose

This section identifies the CMMC Level 1 controls applicable to the organization and describes how the organization satisfies the Federal Acquisition Regulation (FAR) 52.204-21 requirements for protecting Federal Contract Information (FCI).

Controls described in this section apply only to systems, users, and environments defined as in-scope in the Scope and System Boundaries section.

Control Framework

The organization’s CMMC Level 1 compliance is based exclusively on the 17 practices defined in FAR 52.204-21, organized across the following six domains:

  • Access Control (AC)

  • Identification and Authentication (IA)

  • Media Protection (MP)

  • Physical Protection (PE)

  • System and Communications Protection (SC)

  • System and Information Integrity (SI)

Each control is implemented through a combination of:

  • Management-approved policies

  • Documented procedures

  • Technical and administrative safeguards

  • Verifiable evidence

Control Implementation Approach

For each CMMC Level 1 control, the organization maintains:

  • A defined control objective

  • A documented method of implementation

  • Assigned responsibility for enforcement

  • Evidence demonstrating operational effectiveness

Controls are implemented in a manner that is appropriate to the organization’s size, scope, and operational environment and are enforced consistently across all in-scope systems.

Access Control (AC)

The organization limits system access to authorized users, authorized processes, and authorized devices, and restricts access to approved transactions and functions.

Controls are implemented to:

  • Prevent unauthorized system access

  • Enforce least privilege

  • Control the flow of Federal Contract Information

  • Restrict external system access

Applicable Practices:

  • AC.L1-3.1.1

  • AC.L1-3.1.2

  • AC.L1-3.1.20

  • AC.L1-3.1.22

Identification and Authentication (IA)

The organization ensures that users accessing systems handling FCI are uniquely identified and authenticated prior to being granted access.

Controls are implemented to:

  • Assign unique user identities

  • Authenticate users using approved mechanisms

  • Protect authentication credentials

Applicable Practices:

  • IA.L1-3.5.1

  • IA.L1-3.5.2

Media Protection (MP)

The organization protects FCI by ensuring that media containing FCI is sanitized or destroyed prior to disposal or reuse.

Controls are implemented to:

  • Identify media containing FCI

  • Prevent unauthorized disclosure during disposal

Applicable Practices:

  • MP.L1-3.8.3

Physical Protection (PE)

The organization limits physical access to systems and areas containing FCI to authorized individuals.

Controls are implemented to:

  • Restrict physical access to facilities and systems

  • Control and escort visitors

Applicable Practices:

  • PE.L1-3.10.1

  • PE.L1-3.10.3

System and Communications Protection (SC)

The organization protects system boundaries and communications involving FCI from unauthorized access.

Controls are implemented to:

  • Define and protect system boundaries

  • Protect FCI during transmission

Applicable Practices:

  • SC.L1-3.13.1

  • SC.L1-3.13.5

System and Information Integrity (SI)

The organization protects systems handling FCI from malicious code and integrity threats and remediates identified flaws.

Controls are implemented to:

  • Identify and remediate system vulnerabilities

  • Deploy and maintain malicious code protection

  • Monitor systems for security-relevant events

Applicable Practices:

  • SI.L1-3.14.1

  • SI.L1-3.14.2

  • SI.L1-3.14.4

  • SI.L1-3.14.5

Control Evidence

Evidence supporting the implementation of each control is:

  • Documented

  • Retained

  • Available for assessment upon request

Evidence is maintained in accordance with internal documentation and retention requirements.

Control Review

CMMC Level 1 controls are reviewed:

  • At least annually

  • Upon significant system or operational changes

  • Upon changes to contractual requirements involving FCI
